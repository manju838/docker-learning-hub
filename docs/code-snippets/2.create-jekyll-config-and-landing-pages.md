# 1. Setup Jekyll config file inside docs folder
<!-- 'EOF' and EOF mark the beginning and end of the file -->
```bash
cd docs
cat > _config.yml << 'EOF'
title: Docker Learning Hub                           # Site title (appears in browser tab)
description: Master Docker from basics to production # Site description
url: https://YOUR_USERNAME.github.io                 # Site URL
baseurl: /docker-learning-hub                        # Your repository name
search_enabled: true                                 # Enable search
color_scheme: dark                                   # Color scheme (dark, light, auto)
remote_theme: pages-themes/architect@v0.2.0          # Theme to use

# Plugins are required for remote_theme to work
plugins:
  - jekyll-remote-theme
EOF
```

# 2. Create landing page
```bash
cat > index.md << 'EOF'
---
layout: home          # Use homepage layout template
title: Home           # Browser tab title
nav_order: 1          # First item in navigation
---
<!-- This is the landing page for Github pages -->
<!-- YAML front matter is a block of metadata (key-value pairs) placed at the very top of Markdown files, enclosed by triple dashes (---) -->

# Docker Learning Hub
{: .fs-9 }                                              <!-- Jekyll styling (font size 9) -->

Master Docker through hands-on practice
{: .fs-6 .fw-300 }                                      <!-- Font size 6, weight 300 -->

[Get Started](/fundamentals/){: .btn .btn-primary }      <!-- Styles link as a button -->
EOF
```

# 3. Enable GitHub Pages
# Go to repo settings -> Pages -> Source: main branch /docs folder
